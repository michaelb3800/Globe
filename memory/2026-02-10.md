# Memory: Polymarket Trading Bot Development

**Date:** 2026-02-10
**Session:** Polymarket Bot Architecture & MVP Implementation

## Overview
Built a comprehensive Polymarket trading system with three core strategies, strict risk controls, and paper trading simulation.

## Architecture Components

### 1. Data Layer (`src/data/`)
- `polymarket-client.ts` - API client with REST/WebSocket support
- Endpoints: markets, orderbook, trades, orders, positions
- Rate limiting: 2 req/s, burst 10, retry logic

### 2. Risk Layer (`src/risk/`)
- `risk-manager.ts` - Comprehensive risk controls
- Daily loss limit: 3-5% hard stop
- Max drawdown: 8% hard stop
- Per-market: 2% max exposure
- Per-category: 25% max exposure
- Kill switch with auto-trigger on critical violations
- Kelly-based position sizing (fractional 0.25)

### 3. Execution Layer (`src/execution/`)
- `execution-engine.ts` - Smart order routing
- Slippage protection with orderbook depth analysis
- Post-only orders by default
- Maker vs taker decision logic

### 4. Strategies (`src/strategies/`)
- **NO Harvesting:** Targets YES 0-3% markets, buys NO at 97%+
  - Min liquidity: $500
  - Max spread: 1.5%
  - Resolution: <7 days
  
- **Logic Arbitrage:** Implication trading via event graph
  - Correlation-based edge detection
  - Multi-leg arbitrage execution
  
- **Retail Scanner:** Mean-reversion on price dislocations
  - MA deviation detection
  - Volume spike confirmation

### 5. Utilities (`src/utils/`)
- `paper-trader.ts` - Paper trading simulator
- `backtester.ts` - Historical + Monte Carlo simulation
- Slippage modeling

## Design Principles
1. **Paper-first:** All strategies require simulation testing before live
2. **Risk-first:** Kill switch non-negotiable
3. **Edge-first:** Only trade EV+ opportunities
4. **Diversify:** Avoid concentration across correlated outcomes
